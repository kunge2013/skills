# kungeskill 使用手册

## 简介

kungeskill 是一个 Node.js CLI 工具，用于管理 Claude Code 技能（skills）的安装、卸载和更新。通过 symlink 机制将市场中的技能链接到项目的 `.claude/skills/` 目录，实现技能的快速复用和统一管理。

## 安装

```bash
npm install -g kungeskill
```

要求 Node.js >= 18。

## 全局选项

| 选项 | 说明 |
|------|------|
| `--help` / `-h` | 显示帮助信息 |
| `--version` / `-v` | 显示版本号 |

## 命令详解

### 1. kungeskill init

**用途：** 初始化 marketplace 缓存，从 GitHub 克隆技能仓库到本地。

**语法：**

```bash
kungeskill init
```

**内部逻辑：**

- 读取 `~/.kungeskills/config.json` 中的仓库地址和分支
- 将仓库克隆到 `~/.kungeskills/cache/marketplace/`
- 如果已初始化过，直接返回（幂等操作）

**使用场景：**

- 首次安装 kungeskill 后，执行一次拉取远程技能仓库
- 缓存目录被误删后重新初始化

```bash
npm install -g kungeskill
kungeskill init
# ✓ Marketplace cached to ~/.kungeskills/cache/marketplace
```

---

### 2. kungeskill list

**用途：** 列出所有可用技能或已安装的技能。

**语法：**

```bash
kungeskill list              # 列出所有可用技能
kungeskill list --installed  # 只列出当前项目已安装的
```

**选项：**

| 选项 | 说明 |
|------|------|
| `--installed` | 只显示当前项目已安装的技能 |

**内部逻辑：**

- 合并 git cache + bundled plugins 两个来源（去重）
- 按插件分组显示
- `--installed` 时只扫描当前项目 `.claude/skills/` 目录

**使用场景：**

- 查看市场上有哪些技能可以安装
- 确认当前项目安装了哪些技能

```bash
kungeskill list
# PLUGIN             SKILLS
# openspec-workflow  openspec-explore
#                    openspec-propose
#                    openspec-apply-change
#                    openspec-archive-change
# openspec-trace     opst-business-search
#                    opst-code-anysic
# publish            npm-publish

kungeskill list --installed
# Installed skills in this project:
#   openspec-explore
#   npm-publish
```

---

### 3. kungeskill add \<skill\>

**用途：** 将指定技能安装到当前项目（创建 symlink）。

**语法：**

```bash
kungeskill add <skill-name>          # 安装技能
kungeskill add <skill-name> --force  # 强制重装
```

**选项：**

| 选项 | 说明 |
|------|------|
| `--force` | 强制重装（先删除已有安装再重新链接） |

**内部逻辑：**

1. 如果缓存未初始化，自动执行 init
2. 从 git cache 和 bundled plugins 中查找技能
3. 以 `.git` 定位项目根目录，在 `<project>/.claude/skills/` 下创建 symlink
4. Windows 跨驱动器时自动降级为文件复制

**使用场景：**

- 为项目添加一个新的 Claude Code 技能
- 技能更新后强制重装（copy 模式下需要）

```bash
cd my-project
kungeskill add openspec-explore
# ✓ Installed openspec-explore
#   Link: D:\my-project\.claude\skills\openspec-explore
#   Source: ~/.kungeskills/cache/marketplace/plugins/... (symlink)

# 强制重装（适用于 copy 模式或需要刷新时）
kungeskill add npm-publish --force
# ✓ Installed npm-publish
```

**注意事项：**

- 如果 symlink 源和目标不在同一磁盘驱动器（如缓存在 C: 项目在 D:），会自动降级为文件复制模式
- 文件复制模式下更新不会自动同步，需要 `kungeskill add --force <skill>` 手动刷新

---

### 4. kungeskill remove \<skill\>

**用途：** 从当前项目卸载指定技能。

**语法：**

```bash
kungeskill remove <skill-name>
```

**内部逻辑：**

- 检测目标是 symlink 还是 copy 的目录
- symlink → 安全删除链接（不影响源文件）
- 复制的目录 → 递归删除并给出警告
- 不明文件类型 → 拒绝删除（安全保护）

**使用场景：**

- 项目不再需要某个技能
- 清理失效或断裂的技能链接

```bash
kungeskill remove npm-publish
# ✓ Removed npm-publish
```

---

### 5. kungeskill view

**用途：** 查看当前项目已安装技能的健康状态。

**语法：**

```bash
kungeskill view
```

**内部逻辑：**

- 遍历 `.claude/skills/` 下所有条目
- 检测每个 symlink 的目标是否存在、SKILL.md 是否完整
- 输出表格：名称 / 状态 / 目标路径

**状态说明：**

| 状态 | 含义 |
|------|------|
| `✓ ok` | symlink 有效，SKILL.md 存在 |
| `✗ broken` | 目标路径不存在（源被删或 npm 卸载） |
| `⚠ no SKILL.md` | symlink 有效但目标缺少 SKILL.md |

**使用场景：**

- 检查安装的技能是否正常工作
- 发现哪些 symlink 已断裂

```bash
kungeskill view
# NAME              STATUS     TARGET
# openspec-explore  ✓ ok       ~/.kungeskills/cache/.../openspec-explore
# npm-publish       ✗ broken   (target not found)
```

---

### 6. kungeskill update

**用途：** 更新本地 marketplace 缓存（git pull 远程仓库最新内容）。

**语法：**

```bash
kungeskill update
```

**内部逻辑：**

- 对 `~/.kungeskills/cache/marketplace/` 执行 git pull
- 更新 config 中的 lastSync 时间戳
- symlink 模式下项目中的技能自动同步（无需重装）

**使用场景：**

- 远程仓库有新技能或技能内容更新后，拉取最新版本
- 定期执行保持本地缓存与远程同步

```bash
kungeskill update
# ✓ Marketplace updated
# Symlinks automatically point to updated content. No reinstallation needed.
```

**注意事项：**

- 仅对 git cache 有效，需要先执行过 `kungeskill init`
- symlink 安装的技能会自动获取更新内容
- copy 模式安装的技能不会自动更新，需要 `kungeskill add --force <skill>`

---

### 7. kungeskill doctor

**用途：** 诊断整体健康状态，发现并报告问题，给出修复建议。

**语法：**

```bash
kungeskill doctor
```

**检查项目：**

| 检查项 | 说明 |
|--------|------|
| Cache 目录 | 是否存在且为有效 git 仓库 |
| marketplace.json | 是否可解析，包含多少插件和技能 |
| 上次同步时间 | 从 config 读取 lastSync |
| 已安装技能 | 逐个检查 symlink 状态 |

**使用场景：**

- 安装出现异常时排查问题
- 定期检查确保环境健康
- 获取针对性的修复建议

```bash
kungeskill doctor
# Checks:
#   ✓ Cache directory valid
#   ✓ Marketplace manifest valid (3 plugins, 7 skills)
#   Last sync: 2026-06-26T14:30:00.000Z
#
# Project skills:
#   ✓ openspec-explore — ok
#   ✗ npm-publish — broken (target not found)
#
# Summary: 2 total, 1 ok, 1 broken
# Hint: Run "kungeskill remove npm-publish" to clean up broken links.
```

## 典型工作流

### 场景一：首次使用

```bash
# 1. 全局安装
npm install -g kungeskill

# 2. 初始化缓存（首次执行，后续不需要）
kungeskill init

# 3. 查看有哪些技能
kungeskill list

# 4. 为项目安装技能
cd my-project
kungeskill add openspec-explore
kungeskill add npm-publish
```

### 场景二：日常维护

```bash
# 查看当前项目技能状态
kungeskill view

# 远程有更新时同步
kungeskill update

# 不需要的技能移除
kungeskill remove npm-publish
```

### 场景三：问题排查

```bash
# 运行诊断
kungeskill doctor

# 根据提示修复
kungeskill remove broken-skill     # 清理断裂链接
kungeskill init                     # 重建缓存
kungeskill add skill-name --force   # 强制重装
```

### 场景四：跨驱动器（Windows）

```bash
# 缓存在 C: 项目在 D: 时，add 自动降级为 copy
kungeskill add openspec-explore
# ⚠ Cannot create symlink (cross-drive). Falling back to copy mode.
# ✓ Installed openspec-explore (copied)

# 更新后需要手动刷新
kungeskill update
kungeskill add openspec-explore --force
```

## 卸载

```bash
# 移除项目中的技能
kungeskill remove <skill-name>

# 全局卸载 CLI
npm uninstall -g kungeskill

# (可选) 清理本地缓存
rm -rf ~/.kungeskills
```

## 目录结构说明

```
~/.kungeskills/
├── config.json                          # 全局配置（仓库地址、同步时间）
└── cache/
    └── marketplace/                     # git clone 的市场仓库
        ├── .claude-plugin/
        │   └── marketplace.json         # 插件注册清单
        └── plugins/
            ├── openspec-workflow/
            ├── openspec-trace/
            └── publish/

<project>/
└── .claude/
    └── skills/
        ├── openspec-explore -> ~/.kungeskills/cache/.../openspec-explore  (symlink)
        └── npm-publish -> ~/.kungeskills/cache/.../npm-publish            (symlink)
```

## 技能来源优先级

kungeskill 从两个来源查找技能，合并后去重：

| 优先级 | 来源 | 路径 | 说明 |
|--------|------|------|------|
| 1 | Git Cache | `~/.kungeskills/cache/marketplace/` | 通过 init/update 从远程拉取 |
| 2 | Bundled Plugins | `<npm全局路径>/kungeskill/plugins/` | npm 包自带，离线可用 |

同名技能以 git cache 版本优先。git cache 未初始化时自动使用 bundled plugins。
